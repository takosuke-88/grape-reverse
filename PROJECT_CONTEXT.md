# プロジェクト・マスターコンテキスト: grape-reverse

## 1. プロジェクトの定義とビジョン

- **名称**: grape-reverse (グレープリバース)
- **ドメイン**: <https://grape-reverse.com/>
- **定義**: パチスロAタイプ（ジャグラー・ハナハナ等）の解析ツールを核とした、パチスロ総合情報ポータルサイト。
- **将来像**:
  - 特定機種のツールに留まらず、全Aタイプ機種、およびAT機等の解析情報を網羅する「パチスロ情報のワンストップ・プラットフォーム」を目指す。
  - PWA化、ネイティブアプリ化によるオフライン/現場利用の最適化。
  - ユーザーの立ち回りを支援する専門的なコラム（記事コンテンツ）の継続的な発信。

## 2. 技術スタック・インフラ

- **Frontend**: React + Vite + TypeScript
- **Styling**: Tailwind CSS
- **Routing**: react-router-dom
- **Hosting**: Vercel
- **Analytics**: GA4 (SPA特有の遷移検知を `src/components/GoogleAnalytics.tsx` で制御)

## 3. サイト構造とディレクトリ

- **`/src/pages/`**:
  - 機種別解析・ツールページ（カウンター、ボーナス入力、設定判別）
  - コラム・記事一覧および詳細ページ
  - 逆算ツール等の単体機能ページ
- **`/src/components/`**:
  - `Seo.tsx`: 全ページ必須のメタデータ管理コンポーネント。
  - `GoogleAnalytics.tsx`: ルーティング監視・PV送信コンポーネント。
  - 共通UIパーツ（ヘッダー、メニュー、機種選択）。

## 4. 開発における不変の鉄則

### UI/UX：現場至上主義

- **ブラインドタッチの実現**: ツール画面は「ホールで台を打ちながら操作する」ことを前提とする。ボタンは巨大なタップ領域を確保し、視認性よりも「手の感覚」での操作性を優先する。
- **画面分割（スプリットビュー）への適応**: スマホの画面半分（動画視聴中など）での利用を想定したレスポンシブ設計を維持する。
- **破壊的操作の防御**: データリセットなどのアクションは、操作導線から隔離した位置に配置し、必ず確認ダイアログを挟む。

### システム：データ整合性とSEO

- **SPAページ遷移の担保**: ページ遷移時にGA4へのPV送信が漏れないよう、ルーティングの変更を常に監視し、明示的にイベントを発火させる。
- **SEOの自発的実装**: 新規ページ作成時は `Seo.tsx` を用いて、各ページ固有のタイトル、ディスクリプション、カノニカルURLを必ず設定する。
- **情報の継続性**: 既存ページの改修時、設定済みのSEOパラメータ（特にディスクリプション）は、明示的な指示がない限り変更せず維持する。

## 5. ドメイン知識（パチスロ）

- 各機種（ジャグラー、ハナハナ等）の小役名称、配色、リール配列の特性、および独自の判別ロジック（期待値計算）を尊重したUI/データ構造を維持する。
- 「ぶどう逆算」のような専門的な解析ロジックは、ツール内での独立性と連動性を両立させる。
